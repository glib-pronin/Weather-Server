# Weather Backend API

Це Backend-частина застосунку погоди, реалізована за допомогою Django REST Framework.  

Сервер відповідає за:
- реєстрація користувачів;
- авторизацію через JWT-токени;
- підтвердження електронної пошти;
- зберігання профілю користувача;
- надання список країн, що використовується клієнтським застосунком.

> Дані про погоду не проходять через бекенд. Клієнтський застосунок отримує їх безпосередньо із зовнішнього погодного сервісу. Це дозволяє уникнути зайвих запитів до сервера та спрощує архітектуру проєкту.

## 1. Основні можливості

### Реєстрація та авторизація

- реєстрація нових користувачів;
- авторизація за допомогою JWT токенів;
- оновлення Access Token через Refresh Token;
- отримання інформації про поточного користувача.

### Підтвердження електронної пошти

Після реєстрації користувач отримує шестизначний код підтвердження на електронну пошту.

Особливості реалізації:
- код генерується випадковим чином;
- у базі даних зберігається лише його хеш;
- термін дії коду складає 15 хвилин;
- перевіряється строк дії коду перед валідацією;
- відправка листа виконується в окремому потоці для уникнення блокування основного запиту.

### Профіль користувача

Для кожного користувача створюється профіль із додатковими даними:
- країна;
- код країни;
- місто;
- широта;
- довгота.

Ця інформація використовується клієнтським застосунком для роботи з погодними даними та персоналізації користувацького досвіду.

### Робота з країнами

Проєкт містить окремий додаток countries_app, який відповідає за формування та надання списку країн.

Для цього реалізовано власну Django Management Command, яка:
- отримує перелік країн із бібліотеки pycountry;
- локалізує їх назви українською мовою через Babel;
- формує JSON-файл зі списком країн;
- зберігає файл у директорії data.

Приклад запису:
```
{
  "name": "Україна",
  "code": "UA"
}
```

Під час роботи застосунку список країн читається з підготовленого JSON-файлу без звернення до сторонніх сервісів.

## 4. Моделі

### Profile

Зберігає інформацію про місцезнаходження користувача.

Пов’язана з користувачем через:
- OneToOneField → User

|Field|Description|
|----|---------|
|country|Обрана країна|
|country_code|Код обраної країни|
|city|Обране місто|
|lat|Широта обраного міста|
|lng|Довгота обраного міста|

### EmailVerification

Використовується для підтвердження електронної пошти.

Пов’язана з користувачем через:
- OneToOneField → User

|Field|Description|
|----|---------|
|code_hash|Хеш коду|
|expires_at|Термін придаьності коду|

## 5. API Endpoints 

|Method|Endpoint|Description|
|------|--------|-----------|
|POST|/api/register/|Реєстрація нових користувачів|
|POST|/api/login/|Авторизація користувачів та отримання JWT токенів|
|POST|/api/verify-email/|Підтвердження електронної пошти через код|
|POST|/api/refresh/|Оновлення JWT Access Token|
|GET|/api/me/|Отримання даних поточного користувача|
|PATCH|/api/me/|Оновлення профілю користувача|
|GET|/api/countries/|Отримання списку країн|

---
## 6. JWT

Для авторизації використовується Simple JWT.

Налаштування часу життя токенів:
- Access Token — 10 хвилин;
- Refresh Token — 7 днів.

## 8. Запуск проєкту

1. Клонування репозиторію
```
git clone https://github.com/glib-pronin/Weather-Server.git
cd Weather-Server
```

2. Створення віртуального середовища
```
python -m venv venv
source venv/Scripts/activate
```

3. Встановлення залежностей
```
pip install -r requirements.txt
```

4. Виконання міграцій
```
python manage.py migrate
```

5. Запуск серверу
```
python manage.py runserver
```
